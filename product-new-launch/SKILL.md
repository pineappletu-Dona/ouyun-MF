# 欧孕S级市场分析&产品开发 - SOP流程

---

## 第〇章：分析师铁律（强制）

> **此章节为强制规则，任何账号、任何模型执行本 Skill 时必须严格遵守，不得自行变更。**
> 目的：消除因口径自由发挥导致的输出不一致。

### 0.1 统计口径铁律

| 规则编号 | 规则 | 说明 |
|----------|------|------|
| R-01 | **所有占比必须用销量加权** | `维度X的销量占比 = 该维度覆盖的月销量 ÷ 类目总月销量`。禁止用链接数计数算占比 |
| R-02 | **GMV 计算用逐品求和** | `类目月GMV = Σ(单品到手价 × 单品月销量)`。禁止用"均价 × 总销量" |
| R-03 | **均价必须用销量加权** | `品类均价 = Σ(单品价格 × 单品月销量) ÷ 总月销量`。禁止用简单算术平均 |
| R-04 | **品牌数统计需过滤** | 仅统计有明确品牌名的产品。品牌字段为空/"/"未知"/"其他"/"无"/"白牌"的归入"白牌/其他"一类 |

### 0.2 价格带划分铁律

**统一价格带分界线（不可自行调整）**：

| 分界线（到手价） | 区间 |
|------------------|------|
| 0 ~ 60元 | 入门引流区 |
| 60 ~ 100元 | 大众主力区 |
| 100 ~ 150元 | 品质效率区 |
| 150 ~ 200元 | 中高端区 |
| 200元以上 | 高端小众区 |

> 到手价 = 原价 × 0.85。若用户指定其他折扣系数，则到用户指定，但区间分界线不变。

### 0.3 品牌梯队划分铁律

| 梯队 | 划分标准 |
|------|----------|
| T1 | 销量份额 ≥ 8% |
| T2 | 销量份额 3%~8% |
| T3 | 销量份额 1%~3% |
| 长尾 | 销量份额 < 1% |

### 0.4 消费属性判断铁律

| 属性 | 判断标准 | 禁止自行发挥 |
|------|----------|-------------|
| 消费频次 | 查数据源中是否有明确复购数据支撑。孕产妇装/睡袋等**默认为低频/一次性消费**，除非有情报通/生参复购率数据明确证明高频 |
| 复购驱动 | 季节变化 ≠ 复购。孕妇裤换季买新款 = 新客获取，不是老客复购。禁止使用"复购驱动""换季复购"等误导性表述 |
| 品类角色 | 品类拉新 vs 客群复购，必须基于大盘复购率数据判断，不得凭主观推测 |

### 0.5 数据源字段映射铁律

1. **先读数据源所有字段名**，建立「数据源字段 → SOP维度」的映射表
2. 映射表必须在分析前明确输出，不可跳过
3. 一个 SOP 维度只对应一个数据源字段，不可合并/拆分
4. 未标注的属性值统一归入"未标注"类，不计入有效统计基数

### 0.6 输出格式铁律

| 规则 | 要求 |
|------|------|
| R-F01 | **输出格式默认 Excel（.xlsx）**，多 Sheet 按分析板块分表。用户要求 HTML/Markdown 时可额外输出 |
| R-F02 | 每个维度分析后必须附一行「小结」，格式：`小结：[一句话结论]` |
| R-F03 | 关键差异点（如 A类安全标准仅 xx%）需用颜色/加粗标注 |
| R-F04 | 所有百分比保留1位小数，金额保留整数（元），比率保留3位小数 |
| R-F05 | 报告末尾必须包含：SWOT 四象限 + 一页纸行动清单（P0/P1/P2） |

### 0.7 销售模型参数铁律

> ⚠️ 本章参数为唯一权威默认值，SOP 其他位置出现的「示例值」「如」「默认」均以本章为准，有冲突时以本章覆盖。

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 转化率 | 2.5% | 新品首月目标 |
| PPC | ¥2.5 | 直通车点击单价 |
| 付费访客占比 | **0.40**（40%） | `月推广费 = 月均付费访客 × 付费访客占比 × PPC` |
| 退款率 | **0.15~0.30** | 中值0.225。用户有类目数据时取实际值；计算时须同时输出悲观(0.30)/乐观(0.15)两档 |
| 营销费比 | **0.10~0.40** | 起量期0.30~0.40，稳定期0.10~0.15。用户有类目数据时取实际值；计算时须同时输出两档 |
| 扣点 | 5.5% | 天猫标准 |
| 快递费 | ¥2.5/件 | — |
| 备货系数 | 1.3 | 130%备货率 |
| 备货可售天数 | 55天 | — |

### 0.8 定价定位铁律

| 规则 | 定义 |
|------|------|
| R-P01 | **"中端"定价 = 类目到手价 40%~60% 分位区间**。禁止用主观描述 |
| R-P02 | **竞品对标数量 = 3~5 个**，按销量份额从高到低选取，必须覆盖 T1 至少 1 个 + 欧孕目标价格带至少 1 个 |
| R-P03 | **产品痛点/卖点数量 = 3~5 条**，按竞品差评/中评高频词频排序，禁止 AI 自行编造 |

### 0.9 SWOT & 行动清单模板

> R-F05 要求报告末尾包含 SWOT + 行动清单，此处规定统一模板。

**SWOT 四象限**（必须按此结构，每项 ≤ 3 条）：

```
S（优势）          | W（劣势）
──────────────────┼──────────────────
1. 品牌认知        | 1. 无类目权重
2. 安全背书        | 2. 无基础销量
3. 供应链          | 3. 无客群积累
──────────────────┼──────────────────
O（机会）          | T（威胁）
──────────────────┼──────────────────
1. 竞品空白        | 1. 头部品牌壁垒
2. 需求未满足      | 2. 价格战风险
3. 季节窗口        | 3. 流量成本上升
```

- **S 侧**：欧孕品牌自身的优势（品牌认知、供应链、安全背书等）
- **W 侧**：欧孕在新类目的劣势（无权重、无客群、无基础销量等）
- **O 侧**：外部市场机会（竞品空白、需求未满足、季节窗口等）
- **T 侧**：外部威胁（头部品牌壁垒、价格战、流量成本等）

**行动清单**（按优先级分三级）：

| 优先级 | 动作 | 责任人 | 时间节点 | 成功标准 |
|--------|------|--------|----------|----------|
| P0 | 产品打版 | 品类负责人 | 上市前60天 | 样品确认 |
| P1 | 种草内容 | 运营组长 | 上市前30天 | 50篇KOC笔记 |
| P2 | 站外渠道 | 运营组长 | 上市后30天 | 抖音/小红书各1条爆款 |

- **P0**：不做就无法启动的事项
- **P1**：影响首月销量的关键事项
- **P2**：可2个月内补齐的优化项

### 0.10 类目适配声明

> 本 SOP 的表格字段和示例值以睡袋类目为初始模板。**分析其他品类时**：
> - 「产品形态」适配为该品类的核心形态维度（如裤子类目→版型/裤长/腰型，睡袋类目→分腿/一体/防惊跳等）
> - 「填充物类型」为通用字段，所有品类直接使用
> - 品牌、款式编码、标题热词等欧孕专属内容（如 `OY1-25AW-`、`孕安/孕律`）仅作参考，不强制套用
> - **核心分析逻辑和铁律规则全品类通用，不可因品类不同而跳过或修改**

## 第一章：SOP流程总览

### 1.1 六步流程图

```
Step 1 ─→ Step 2 ─→ Step 3 ─→ Step 4 ─→ Step 5 ─→ Step 6
品类定位    大盘采集    大盘分析    竞品分析    定价定级    开品需求
                                                       &布局
```

| Step | 名称 | 责权人 | 核心产出 | 前置依赖 |
|------|------|--------|----------|----------|
| **1** | 品类定位 | 品类负责人 | 5维定位结论 + 品类分级 | — |
| **2** | 大盘数据采集 | 运营助理 | TOP100数据 + 搜索词 + 大盘占比 | Step 1 |
| **3** | 大盘分析 | 运营组长 | 品牌份额 + 多维度透视 + 价格带 + 机会点 | Step 2 |
| **4** | 竞品深度分析 | 运营组长+助理 | 竞品对标表 + 评价提炼 + 站外布局 | Step 3 |
| **5** | 定价定级&销售模型 | 运营组长 | 定价表 + 销售模型 + 月度拆解 | Step 4 |
| **6** | 开品需求&产品布局 | 品类负责人+组长 | SKU表 + SPU表 + 产品布局 + 营销策略 | Step 5 |

> **要领**：严格按 Step 1→6 顺序执行，每个 Step 的输出是下一个 Step 的输入。

### 1.2 岗位责权

```
品类负责人：Step 1（定位）→ Step 6（开品需求确认）
运营组长：  Step 3（大盘分析）→ Step 4（竞品分析）→ Step 5（定价定级）
运营助理：  Step 2（数据采集）→ Step 4（评价提炼）→ Step 6（数据支持）
```

### 1.3 品类分级

| 级别 | 走哪些步骤 | 简化规则 |
|------|-----------|----------|
| **S/S+级** | 全部6步，无省略 | — |
| **A级** | Step 1~5 + Step 6 部分 | Step 6 可简化，不做精细化 SKU 色系/设计参考 |
| **B级以下** | Step 1~4 + Step 5 简化 | 仅做大盘概况 + 核心竞品分析 + 定价参考 |

## 第二章：Step 1 - 品类定位

### 要点说明

品类定位是整个 SOP 的**起点**，决定了后续所有分析的方向。定位错误会导致全盘浪费。

**核心问题**：这个品类值不值得做？以什么角色切入？

### 1.1 五维定位清单

| 序号 | 维度 | 核心问题 | 决策影响 |
|------|------|----------|----------|
| 1 | 类目角色 | 品类拉新 vs 客群复购？ | 决定营销预算分配方向（见铁律0.4） |
| 2 | 季节属性 | 产品线季节性如何？ | 决定开发节奏和备货周期 |
| 3 | 消费属性 | 消费频次如何？（低频/一次性/中频/高频） | 决定 SKU 数量和价格带宽度 |
| 4 | 客群年龄 | 品牌客群月龄段分布？ | 预警：出生率&抗风险能力 |
| 5 | 消费频次 | 产品消费周期多长？ | 数据化客群画像模型预估 |

### 1.2 大盘概况数据源

| 序号 | 大盘板块 | 数据源 |
|------|----------|--------|
| 一 | 婴童大盘 | 情报通BI + 生参 |
| 二 | 孕产大盘 | 情报通BI + 生参 |
| 三 | 亲子装大盘 | 情报通BI + 生参 |
| 四 | 亲子鞋大盘 | 情报通BI + 生参 |
| 五 | 婴童尿裤大盘 | 情报通BI + 生参 |
| 六 | 婴童洗护大盘 | 情报通BI + 生参 |

> 各品类大盘数据可跨品类共用。

## 第三章：Step 2 - 大盘数据采集

### 要点说明

数据采集是分析的**原材料**。数据质量直接决定分析结论的可靠性。

**核心原则**：先采集原始数据，再加工分析。禁止跳过采集直接凭经验输出结论。

### 3.1 模块8：单链主图采集（TOP50）

| 字段 | 说明 | 采集方式 |
|------|------|----------|
| 单链排名 | TOP排名 | 情报通BI |
| 单链地址 | tmall链接（仅ID，不直接访问） | 情报通BI |
| 品牌 | 品牌名 | — |
| 主图1~5 | 5张主图截图 | Snipaste + 浏览器切图 |
| 核心卖点 | 主图文案提炼 | 人工提炼 |
| 面料 | 产品材质 | 数据源字段 |
| 成交均价 | 销售额/销量 | 计算 |
| 价格带 | 按铁律0.2划分 | 计算 |
| 近一年总销量 | 年度销量 | 数据源 |
| 近一年销售额 | 年度销售额 | 数据源 |
| 填充物类型 | 填充材质 | 数据源字段（通用） |
| 外包装 | 包装类型 | 数据源字段 |
| 产品形态 | 按该品类核心形态维度（见0.10） | 数据源字段 |
| 季节选款需求 | 春夏/四季/秋冬 | 数据源字段 |

> ⚠️ 通过生参后台输入商品ID采集，**不要直接点击链接**（避免违规）。

### 3.2 模块9：搜索排行采集（类目大词）

| 字段 | 说明 |
|------|------|
| 统计时间 | 近30日区间 |
| 排名 | 搜索词排名 |
| 相关搜索词 | 类目大词/长尾词 |
| 搜索人气 | 搜索量级 |
| 点击率 | CTR |

| 30日最低访客数据 | 最低UV |
| 最低日均访客 | 日均UV |

### 3.3 模块10：单品拆解（TOP50，43列）

| **品牌信息** | 品牌、掌柜、掌柜链接、店铺名称、地域、时间维度 |

> **数据源**：情报通BI 直接下载。单链产品属性只能手动拆解，情报通仅提供大盘产品属性。

### 3.4 产品属性后台采集

- WPS 共用账号（企业版VIP）采集
- TOP100 大盘采集后需做透视小结
- 核心大指标 + 各明细小指标需分别提取

## 第四章：Step 3 - 大盘分析

### 要点说明

大盘分析的目标是**看清市场全貌**：谁在卖、卖多少、什么价格、什么形态。分析结果直接驱动 Step 4 的竞品选择和 Step 5 的定价决策。

**核心产出**：品牌份额表 + 多维度透视表 + 价格带机会表 + 市场机会点。

### 4.1 A3：市场机会-大指标（TOP100透视）

**表头字段**：

| 字段 | 说明 |
|------|------|
| 品牌 | TOP品牌名 |
| 产品形态 | 按数据源实际字段（如分腿/一体/紧身/宽松/直筒等） |
| 价格带 | 按铁律0.2统一分界线划分 |
| 面料 | 按数据源实际字段 |
| 填充物类型 | 按数据源实际字段（通用字段） |
| 宝贝链接 | 单链地址 |
| 上架月份 | 上市时间 |
| 货号 | 产品编码 |
| 销量 | 近一年总销量 |
| 销售额 | 近一年总销售额 |
| 成交均价 | 总销售额/总销量 |
| 淘系三级类目TOP100上榜链接数 | 上榜数量 |

### 4.2 A4：市场机会-多指标（六维交叉透视）

**核心产出**：多维度交叉透视（TOP100 × 面料 × 产品形态 × 包装 × 场景 × 年龄段）

| 板块 | 维度1 | 维度2 | 核心指标 |
|------|-------|-------|----------|
| TOP100大盘 | 品牌 | 销售额/销量 | 品牌份额 |
| TOP100面料 | 品牌 × 价格带 × 面料 × 填充物 | 销量/销售额 | 材质偏好 |
| 产品形态 × 客群月龄 | 品牌 × 形态 × 月龄段 | 销量/销售额 | 形态-客群匹配 |
| 包装 × 卖点 | 品牌 × 包装 × 形态 × 功能 × 价格带 | 销量/销售额/成交均价 | 包装策略 |
| 外包装 × 品类卖点 | 品牌 × 包装 × 形态 × 功能 × 价格带 | 销量/销售额/成交均价 | 卖点提炼 |
| 适用场景 × SKU月龄 × 季节 | 品牌 × 场景 × 季节 × 年龄段 × 功能 × 长度 | 销量/销售额 | 季节布局 |

> **数据源**：情报通BI — TOP100单链市场排名（淘系三级类目，25年全年）

## 第五章：Step 4 - 竞品深度分析

### 要点说明

竞品分析不是列名单，是**找差异化切入点**。重点是：竞品哪里做得好（对标）、哪里做得不好（机会）。

**核心产出**：竞品购买清单 + 竞品对标表 + 评价痛点提炼 + 站外布局分析。

### 5.1 B4：竞品购买（实物对标）

| 字段 | 说明 |
|------|------|
| 竞品购买序号 | 1~11 |
| 品牌 | 竞品品牌名 |
| 产品首图 | 截图 |

## 第六章：Step 5 - 定价定级与销售模型

### 要点说明

定价定级是**从分析到决策的关键转折点**。前面的分析全部是"看清市场"，这一步是"确定打法"。

**核心原则**：定价基于竞品对标，销售模型基于保守预估，不乐观假设。

### 6.1 模块1：定价定级

#### 定价四大原则

| 原则 | 内容 |
|------|------|
| 原则1 | 整体价格定位中端（= 类目到手价 40%~60% 分位区间，见铁律0.8 R-P01） |
| 原则2 | 定价确认后一般不允许破价；特殊破价需提前申请 |
| 原则3 | A级和S级活动采用立减金额方式，确保日常/A级/S级渠道价拉通 |
| 原则4 | 淘客/挑客以单链接1件为主，2件及以上需另行申请 |

#### 定价表结构（含公式）

| 字段 | 公式/说明 |
|------|-----------|
| 序列 | 1/2/3... |
| 件数 | 1件/3条包装等 |
| 产品名称 | SKU名称 |
| 产品定级 | S/A/B |
| 到手价\|倍率 | `= 前台价 - 优惠券` |
| 款式编码 | OY1-25AW-XXXXX-X（欧孕品牌专属参考） |
| 颜色 | 颜色SKU |
| 规格 | 如 80-130 |
| 成本价 | 采购/生产成本 |
| 前台价 | 标价 |
| 优惠券 | 券面额 |
| **日常到手价** | **`= 前台价 - 优惠券`** |
| **日常扣点** | **`= 到手价 × 5.5%`** |
| 快递费 | 固定值（见铁律0.7） |
| **日常毛利额** | **`= 到手价 × (1-退款率) - 扣点 - 快递费 - 成本价 - 到手价 × 营销费比`** |
| **日常毛利率** | **`= 毛利额 / 到手价`** |
| 退款率 | **0.15~0.30**（见铁律0.7） |
| 营销费比 | **起量期0.30~0.40 / 稳定期0.10~0.15**（见铁律0.7） |
| **实际保本ROI** | **`= 1 / 毛利率`** |

#### 活动层级定价

| 层级 | 折扣系数 | 到手价公式 |
|------|----------|------------|
| 日常 | 1.0 | 前台价 - 优惠券 |
| A级活动 | 0.9 | 前台价 × 0.9 - 优惠券 |
| A+级活动 | 0.88 | 前台价 × 0.88 - 优惠券 |
| S级活动 | 0.85 | 前台价 × 0.85 - 优惠券 |

> **公式模板**（以A级为例）：
> ```
> A级到手价 = 前台价 × 0.9 - 优惠券
> A级扣点   = A级到手价 × 5.5%
> A级毛利额 = A级到手价 × (1-退款率) - 扣点 - 快递费 - 成本 - 到手价 × 营销费比
> A级毛利率 = A级毛利额 / A级到手价
> 保本ROI   = 1 / 毛利率
> ```

#### 竞品对标区

> 竞品选择规则（见铁律0.8 R-P02）：3~5个，按销量份额从高到低，必须覆盖 T1 至少 1 个 + 欧孕目标价格带至少 1 个。

| 阶段 | 公式 | 示例 |
|------|------|------|

#### 月度拆解公式

```
月销售额      = 年度目标 × 对应月份大盘占比
月推广费      = 月均付费访客 × 付费访客占比 × PPC
月需访客      = 月销售额 / 客单价 / 转化率
月均单量      = 月销售额 / 客单价
月均付费访客  = 月需访客 × 付费访客占比
日销售额      = 月销售额 / 30
日推广费      = 月推广费 / 30
日均单量      = 月均单量 / 30
日均付费访客  = 月均付费访客 / 30
```

#### 产品库存成本

```
产品库存成本 = 月销售额 × 1.3 / 30 × 备货可售天数
```

#### 月度经营指标

| 指标 | 公式 |
|------|------|

| 月度毛利润占比 | `= 1 - 退款率 - 营销费比 - 扣点`（按0.7参数推导，禁止使用硬编码值） |

## 附录A：核心公式速查

### A.1 定价类公式

> 参数默认值见铁律0.7。

```
到手价     = 前台价 × 折扣系数 - 优惠券
扣点       = 到手价 × 5.5%
毛利额     = 到手价 × (1 - 退款率) - 扣点 - 快递费 - 成本价 - 到手价 × 营销费比
毛利率     = 毛利额 / 到手价
保本ROI    = 1 / 毛利率
定价倍率   = 产品定价 / 成本需求
```

### A.2 销售模型公式

```
月销售额      = 年度目标 × 月大盘占比
月推广费      = 月均付费访客 × 付费访客占比 × PPC
月需访客      = 月销售额 / 客单价 / 转化率
月均单量      = 月销售额 / 客单价
产品库存成本  = 月销售额 × 1.3 / 30 × 备货天数
月度营销费比  = 月推广费 / 月销售额
月度毛利润    = 月销售额 × 毛利率
```

### A.3 产品布局公式

```
定价倍率          = 价格带平均件单价 / 预估成本
全渠道总月销      = SUM(天猫 + 京东 + 拼多多 + 抖音各渠道)
销售额预估        = 全渠道总月销 × 价格带平均件单价
月度平均库存货值  = (全渠道总月销 × 1.3 × 均价 / 30) × 备货可售天数
营销费用预估      = 销售额预估 × 营销费比（见铁律0.7分阶段取值）
```

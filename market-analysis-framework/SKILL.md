# S级产品市场分析通用框架

> 适用：淘系/抖音/京东全品类
> 定位：纯方法论 + 铁律 + 公式，不含品类具体数据
> 被引用：`product-new-launch`（纯新品开发）、`product-legacy-upgrade`（老品升级）

---

## 第〇章：分析师铁律（强制）

> **强制规则，任何账号、任何模型执行本 Skill 时必须严格遵守。**

### 0.1 统计口径铁律

| 规则编号 | 规则 | 说明 |
|----------|------|------|
| R-01 | **所有占比必须用销量加权** | `维度X的销量占比 = 该维度覆盖的月销量 ÷ 类目总月销量`。禁止用链接数计数算占比 |
| R-02 | **GMV 计算用逐品求和** | `类目月GMV = Σ(单品到手价 × 单品月销量)`。禁止用"均价 × 总销量" |
| R-03 | **均价必须用销量加权** | `品类均价 = Σ(单品价格 × 单品月销量) ÷ 总月销量`。禁止用简单算术平均 |
| R-04 | **品牌数统计需过滤** | 仅统计有明确品牌名的产品。品牌字段为空/"/"未知"/"其他"/"无"/"白牌"的归入"白牌/其他"一类 |

### 0.2 价格带划分铁律

**统一价格带分界线（不可自行调整）**：

| 分界线（到手价） | 区间 |
|------------------|------|
| 0 ~ 60元 | 入门引流区 |
| 60 ~ 100元 | 大众主力区 |
| 100 ~ 150元 | 品质效率区 |
| 150 ~ 200元 | 中高端区 |
| 200元以上 | 高端小众区 |

> 到手价 = 原价 × 0.85。若用户指定其他折扣系数，则到用户指定，但区间分界线不变。

### 0.3 品牌梯队划分铁律

| 梯队 | 划分标准 |
|------|----------|
| T1 | 销量份额 ≥ 8% |
| T2 | 销量份额 3%~8% |
| T3 | 销量份额 1%~3% |
| 长尾 | 销量份额 < 1% |

### 0.4 消费属性判断铁律

| 属性 | 判断标准 | 禁止自行发挥 |
|------|----------|-------------|
| 消费频次 | 查数据源中是否有明确复购数据支撑。孕产妇装/睡袋等**默认为低频/一次性消费**，除非有情报通/生参复购率数据明确证明高频 |
| 复购驱动 | 季节变化 ≠ 复购。换季买新款 = 新客获取，不是老客复购。禁止使用"复购驱动""换季复购"等误导性表述 |
| 品类角色 | 品类拉新 vs 客群复购，必须基于大盘复购率数据判断，不得凭主观推测 |

### 0.5 输出格式铁律

| 规则 | 要求 |
|------|------|
| R-F01 | **输出格式默认 Excel（.xlsx）**，多 Sheet 按分析板块分表。用户要求 HTML/Markdown 时可额外输出 |
| R-F02 | 每个维度分析后必须附一行「小结」，格式：`小结：[一句话结论]` |
| R-F03 | 关键差异点需用颜色/加粗标注 |
| R-F04 | 所有百分比保留1位小数，金额保留整数（元），比率保留3位小数 |
| R-F05 | 报告末尾必须包含：SWOT 四象限 + 一页纸行动清单（P0/P1/P2） |

### 0.6 销售模型参数铁律

> ⚠️ 本章参数为唯一权威默认值，有冲突时以本章为准。

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 转化率 | 2.5% | 新品首月目标 |
| PPC | ¥2.5 | 直通车点击单价 |
| 付费访客占比 | **0.40**（40%） | `月推广费 = 月均付费访客 × 付费访客占比 × PPC` |
| 退款率 | **0.15~0.30** | 中值0.225。有类目数据时取实际值；计算时须同时输出悲观(0.30)/乐观(0.15)两档 |
| 营销费比 | **0.10~0.40** | 起量期0.30~0.40，稳定期0.10~0.15。有类目数据时取实际值 |
| 扣点 | 5.5% | 天猫标准 |
| 快递费 | ¥2.5/件 | — |
| 备货系数 | 1.3 | 130%备货率 |
| 备货可售天数 | 55天 | — |

### 0.7 定价定位铁律

| 规则 | 定义 |
|------|------|
| R-P01 | **"中端"定价 = 类目到手价 40%~60% 分位区间**。禁止用主观描述 |
| R-P02 | **竞品对标数量 = 3~5 个**，按销量份额从高到低选取，必须覆盖 T1 至少 1 个 + 目标价格带至少 1 个 |
| R-P03 | **产品痛点/卖点数量 = 3~5 条**，按竞品差评/中评高频词频排序，禁止 AI 自行编造 |

## 一、价格带分析

### 1.1 标准流程

1. 获取类目 TOP100~200 单链数据（到手价 + 月销量）
2. 按 5 段分界线（60/100/150/200）划分区间
3. 计算每个区间：链接数量、销量占比、销售额占比、均价
4. 输出 Excel 价格带分布表

### 1.2 AI 话术模板

```
请按以下分界线划分价格带（到手价 = 原价 × 0.85）：

| 分界线 | 区间 |
|--------|------|
| 0 ~ 60元 | 入门引流区 |
| 60 ~ 100元 | 大众主力区 |
| 100 ~ 150元 | 品质效率区 |
| 150 ~ 200元 | 中高端区 |
| 200元以上 | 高端小众区 |

请输出每个价格带的：链接数量、销量占比、销售额占比、均价
```

### 1.3 分位数价格带（进阶）

当需要更精细的价格带分析时，使用分位数法：

```
POST /api/price-band-analysis
Body: { "itemIds": [...], "percentiles": [10,25,50,75,90] }
```

> 完整 API 参数见 **附录B：tmall-scraper 联动**。

## 三、定价定级

### 3.1 三层定价公式

| 层级 | 定义 | 计算公式 |
|------|------|----------|
| **日常价** | 日常销售到手价 | `成本 × (1 + 目标毛利率) + 快递费` |
| **A级活动价** | 618/99/双11等大促价 | `日常价 × 0.85`（扣点+平台满减后综合折扣） |
| **S级活动价** | 双11/双12等超级大促价 | `日常价 × 0.75`（平台要求最低折扣） |

### 3.2 销售模型公式

**核心公式**：

```
月销量目标 = 目标GMV ÷ 到手价
月访客数 = 月销量目标 ÷ 转化率
月付费访客 = 月访客数 × 付费访客占比
月推广费 = 月付费访客 × PPC
```

**盈亏测算**：

```
月毛利额 = 目标GMV × (1 - 退款率) × (1 - 扣点)
月营销费 = 目标GMV × 营销费比
月快递费 = 月销量目标 × 快递费单价
月净利润 = 月毛利额 - 月营销费 - 月快递费 - 月固定成本
```

### 3.3 备货公式

```
首单备货量 = 月销量目标 × 备货系数 × (备货可售天数 ÷ 30)
```

## 四、营销策略

### 4.1 渠道分类

| 类型 | 渠道 | 特点 |
|------|------|------|
| **一类渠道** | 淘系、京东、拼多多 | 收割+品牌曝光，搜索流量为主 |
| **二类渠道** | 抖音自播/达播/短视频/商品卡、快手、小红书 | 种草+收割一体，内容驱动 |

### 4.2 入局门槛分析

| 分析维度 | 一类渠道 | 二类渠道 |
|----------|----------|----------|
| 竞品流量结构 | 自然流/付费占比 | 短视频/自播/达播/商品卡占比 |
| 付费入局门槛 | 直通车PPC × 目标访客 | 达人佣金 + 投流费 |
| 内容形式 | 主图+详情页 | 产品卖点/实测/开箱对比 |
| 转化入口 | 搜索直达 | 搜索直达/直播间/评论区 |

### 4.3 盈亏测算模板

```
品类：[品类名]
目标客群：[DMP人群画像]
月度营销目标：
  - 一类渠道：GMV XX万，费用比 XX%
  - 二类渠道：GMV XX万，费用比 XX%
综合盈亏：
  - 一类毛利额：XX
  - 二类毛利额：XX
  - 综合净利润：XX
```

### 4.4 AI 话术模板

```
请分析竞品的站内外营销布局：
1. 一类渠道（淘系/京东/拼多多）：
   - 竞品流量结构（自然流/付费占比）
   - 付费入局门槛预估
2. 二类渠道（抖音/快手/小红书）：
   - 竞品渠道销量占比（短视频/自播/达播/商品卡）
   - 内容形式（产品卖点/实测/开箱对比）
   - 转化入口（搜索直达/直播间/评论区）
3. 给新品入局的营销策略建议（含预算预估）
```

## 附录D：AI分析能力边界

> 来源：XMind「AI-结论→优势/劣势」

### D.1 优势

- 加载 Skill 到 WorkBuddy，输入提示词&需求词，**5-10分钟**可覆盖分析 **100-300个**淘系品类TOP排行榜
- 现有 S 级市场分析 Skill 可覆盖 **95%** 需求
- 标准化输出格式，跨账号一致性高

### D.2 劣势

- 无法一次性优化到位，全品类所有需求无法全精准覆盖
- 结论需**人工修正**，AI分析结果仅作为初稿参考
- 部分工具（Codex/Claude Code/OpenClaw）需VPN或复杂配置

### D.3 使用原则

1. AI 分析结论 = 初稿，必须经品类负责人人工复核
2. 关键数据（成本、销量目标、定价）以人工确认为准
3. 颜色/花型/款式方向必须基于数据源TOP值，AI仅做统计排序

# 老品升级标准流程

> 老品升级 ≠ 从零开品。核心差异在于**已有销售数据 + 已有客群认知**，升级目标是**优化现有链接/扩展新款**而非全新品类定位。
> 方法论基础见 `market-analysis-framework` Skill（铁律+公式）。

---

## 第〇章：老品升级铁律（精简版）

> 以下为老品升级适用的强制规则，完整铁律见 `market-analysis-framework` §0。

| 规则编号 | 规则 | 说明 |
|----------|------|------|
| R-01 | **所有占比必须用销量加权** | 禁止用链接数计数算占比 |
| R-02 | **GMV 计算用逐品求和** | 禁止用"均价 × 总销量" |
| R-03 | **均价必须用销量加权** | 禁止用简单算术平均 |
| R-04 | **到手价 = 原价 × 0.85** | 统一折扣系数 |

### 价格带拆分标准（老品升级专用）

| 价格区间 | 步长 | 说明 |
|----------|------|------|
| 50元以下 | 5元 | 低客单快消，密集拆分 |
| 50~100元 | 15元 | 中低客单 |
| 100~150元 | 20元 | 中客单核心区间 |
| 150~200元 | 35元 | 中高客单 |
| 200元以上 | 50元 | 高客单，宽松拆分 |

## 一、老品升级 vs 新品开发 差异对照

| 维度 | 新品开发 | 老品升级 |
|------|----------|----------|
| 数据基础 | 大盘数据 + 竞品推测 | 自有销售数据 + 评价数据 + 竞品数据 |
| 客群认知 | DMP画像推断 | 已成交客群精准画像 |
| 核心问题 | "这个品类能不能做" | "现有品怎么做更好/怎么扩款" |
| 分析重点 | 市场容量 + 价格带缺口 | 评价差评归因 + 竞品变化 + 价格带迁移 |
| 价格策略 | 定价模型反推 | 现有价格带优化 + 上下延伸 |

## 二、老品升级 6 步流程

```
情报通/生参 → 竞品链接提取 → tmall-scraper采集 → 插件提炼 → AI话术分析 → 升级结论
```

| 步骤 | 操作 | 工具 | 输出 |
|------|------|------|------|
| **1. 竞品链接提取** | 情报通/生参提取品类 TOP100 竞品链接 | 情报通BI / 生参 | 竞品链接清单.csv |
| **2. 价格带+款式拆分** | 按售价拆分价格带，按主图人工分款式 | Excel/WPS | 分类后竞品链接清单 |
| **3. 天猫数据采集** | 基础信息+主图+评价+问大家批量采集 | tmall-scraper（见附录B） | 采集结果表 |
| **4. 插件提炼** | 欧孕插件提取核心属性（面料/形态/场景/月龄/季节/填充物/SKU/包装/标题卖点） | tmall-scraper 插件 | 属性提炼表 |
| **5. AI话术分析** | 评价分析 + 主图分析 + 属性分析 | WorkBuddy + 通义千问 | AI分析报告 |
| **6. 升级结论** | 切入价格带/需求成本/核心属性/入局门槛/货盘成本/营销预算 | 人工复核 | 老品升级方案 |

## 三、款式人工筛选规则

> ⚠️ **关键步骤**：按主图进行款式分类，比按标题分类更精准。

1. 打开采集结果表，逐行查看主图
2. 按**产品形态**归入款式类别（如：一体式睡袋/分腿睡袋/背心式睡袋）
3. 同一款式类别内，再按**花色/图案**细分
4. 标记异常链接（SKU混乱、主图与标题不符）
5. 剔除不相关链接（非目标品类）

## 四、6 大核心分析维度

| 维度 | 分析方法 | 数据来源 |
|------|----------|----------|
| **差评归因** | 提取竞品+自有链接差评，分类统计（面料/尺寸/掉色/起球/拉链…） | 评价采集 |
| **SKU销售占比** | 各颜色/尺寸 SKU 销量占比，指导新款 SKU 布局 | 评价采集 SKU 维度 |
| **价格带迁移** | 对比去年同期价格带分布，判断消费升级/降级趋势 | 情报通/生参 |
| **主图卖点演进** | 对比新旧链接主图卖点变化，判断市场风向 | 主图采集+AI分析 |
| **对标品牌变化** | 去年同期 vs 今年 TOP 品牌排名变化 | 情报通品牌排名 |
| **问大家热点** | 提取高频问题，定位消费者决策疑虑 | 问大家采集 |

## 五、AI 话术模板

### 5.1 竞品链接评价分析

**适用场景**：已有竞品链接，需要分析评价来提炼产品优化方向

```
请分析以下竞品链接的评价数据：
1. 对应品牌链接差评分析、好评分析
2. 根据每个品牌链接SKU颜色销售占比、尺寸销售占比，
   并给出产品开发花型颜色设计方向以及尺码布局建议
3. 通过评价，新品开发需要优化的地方
```

**数据来源**：`POST /api/review/analyze-batch`

---

### 5.2 竞品链接属性/参数分析

**适用场景**：需要从竞品链接中提取产品属性，辅助开品决策

```
请分析以下竞品链接：
1. 产品属性分析-面料，[产品形态]款式，适用场景，年龄段，季节，填充物
2. SKU维度--需要进行月龄段拆解，颜色分类
3. 包装种类，进行分析
4. 产品标题--需要拆解竞品买的&卖点
5. 给出款式开发方向以及功能差异化卖点的开发方向
6. 上市年份时间，安全类别
```

> ⚠️ `[产品形态]` 需按品类替换，见品类适配表

**数据来源**：`POST /api/scrape`（pro模式）

---

### 5.3 竞品主图视觉分析

**适用场景**：需要拆解竞品主图的视觉策略，提炼拍摄/设计方向

```
请分析竞品主图：
1. 产品核心款式形态/核心差异化卖点/价格锚点 & 营销利益点/拍摄场景
2. 面料成分/面料物理属性展示/环保属性/克重参数
3. 缝制工艺/辅料细节/人性化细节设计
4. 适用场景/sku 组合方式/尺码表信息/花色组合
5. 资质背书/售后保障/包装规格/附加服务
```

**数据来源**：`POST /api/scrape` → `main_images`（5张主图）

## 附录B：tmall-scraper 调用流程

### B.1 采集顺序

```
POST /api/scrape              → 批量采集基础信息+主图
POST /api/review/analyze-batch → 批量评价采集+SKU占比+商品对标
POST /api/qa/analyze-batch     → 批量问大家采集
POST /api/price-band-analysis  → 分位数价格带分析
POST /api/price-band-analysis/insights → AI 智能结论
```

### B.2 字段映射

| 话术类别 | 所需数据 | tmall-scraper API |
|----------|----------|-------------------|
| 竞品评价分析 | 评价数据（差评/好评/SKU占比） | `POST /api/review/analyze-batch` |
| 竞品属性分析 | 商品属性（面料/形态/场景/季节等） | `POST /api/scrape`（pro模式） |
| 竞品主图分析 | 主图5张 | `POST /api/scrape`（返回`main_images`） |
| 价格带分析 | 商品价格列表 | `POST /api/price-band-analysis` |

### B.3 联动流程

```
情报通/生参（TOP100链接）→ 手动导入CSV
        ↓
tmall-scraper（单品+评价+问大家+价格）→ API自动调用
        ↓
通义千问（AI分析结论）→ 填入话术模板分析
```
